# L3 复杂需求 — 分析模板

当需求被评定为 L3 等级时，读取本文件执行完整分析。L3 在 L2 可行性评估的基础上，增加依赖关系图和按需业界调研。

## 可行性 + 影响评估

使用 `analysis-l2.md` 中的可行性评估表格（L2 和 L3 共用同一个表格格式），然后继续以下内容。

## 需求依赖关系

将所有需求的依赖关系绘制成全局依赖图：

```
需求依赖图 (实线=前置依赖, 虚线=关联影响):

  ┌──────────┐     prerequisite     ┌──────────┐
  │  需求 A   │ ───────────────────▶│  需求 B   │
  │  用户认证  │                     │  权限管理  │
  └─────┬────┘                     └─────┬────┘
        │                                │
        │  - - - - 关联影响 - - - -       │ prerequisite
        │                                │
        │          ┌──────────┐          │
        └─────────▶│  需求 C   │◀─────────┘
                   │  审计日志  │
                   └──────────┘

  ┌──────────┐
  │  需求 D   │  (独立，无依赖，可并行开发)
  │  导出报表  │
  └──────────┘
```

在依赖图下方附上文字说明：

- **前置依赖**：B 依赖 A 先完成，A 的接口设计会影响 B 的实现
- **关联影响**：A 的数据模型变更会影响 C 的查询逻辑，需要协调
- **独立需求**：D 无依赖，可随时开发
- **建议开发顺序**：A → B → C，D 可并行

## 业界实现方案调研

**仅在涉及技术选型时执行。** 如果 L3 需求不涉及选型（如纯架构重构、流程改造），跳过此调研，在可行性评估中说清楚方案思路即可。

当 L3 需求的核心问题涉及"选哪个技术/框架/方案"时，给出业界常见方案的对比：
